# Accessibility

BusTerminal's foundation is engineered so accessibility is a property of
the primitives, not an audit run before release. Every primitive ships
keyboard-operable, screen-reader-labeled, and `vitest-axe`-clean at
WCAG 2.2 AA. The behaviors documented on this page are **contracts** —
breaking them is a defect, not a stylistic choice.

---

## Conformance target

- **WCAG 2.2 AA** minimum across every primitive and composite
  (FR-027 / SC-002 / SC-005).
- **WCAG 2.2 AAA** where practical (e.g., contrast pairings that already
  exceed AA are recorded as AAA).
- Continuous enforcement:
  - `vitest-axe` runs in every primitive's unit test.
  - `@storybook/test-runner` + `axe-playwright` scans every published
    story (`pnpm test:storybook`) with the zero-violation threshold
    configured in `.storybook/test-runner.ts` (T107).
  - Story-runner rule set: `wcag2a`, `wcag2aa`, `wcag21a`, `wcag21aa`,
    `wcag22aa`.

---

## Keyboard contract

Every interaction must complete without a pointing device (SC-007). The
Playwright spec `tests/e2e/keyboard-only.spec.ts` (T109) walks the
representative composed screen end-to-end with `page.keyboard.*` only.

### Focus indication

- A visible focus ring is required on every focusable element. The
  global `:focus-visible` rule in `app/globals.css` paints an outline
  using the design tokens `--focus-ring-width`, `--focus-ring-color`,
  and `--focus-ring-offset` — primitives inherit it for free.
- Primitives may layer in additional `focus-visible:outline-*` utilities
  to override offset / color in surfaces where the default would clash
  (e.g., a button on an accent surface), but they must NOT remove the
  focus ring without a documented replacement.

### Tab order

- Tab order follows the document order. Use `tabIndex={0}` only to
  promote a non-interactive container that needs to be reachable (e.g.,
  a scrollable table region with keyboard navigation).
- Do not use `tabIndex` ≥ 1. Manually-ordered tab sequences are
  fragile and conflict with assistive technologies that re-flow based
  on the accessibility tree.

### Activation

| Element | Activation key |
|---|---|
| Button | `Enter` or `Space` |
| Toggle / Switch | `Space` |
| Tab trigger | `Arrow` keys to move focus, `Enter` to select (Radix default) |
| Menu item | `Arrow` keys to navigate, `Enter` to invoke |
| Combobox / Command palette | `Cmd/Ctrl+K` to open, `Arrow` keys + `Enter` to commit |
| Dialog / Sheet close | `Escape` |
| Row selection (DataTable) | `Space` |
| Row navigation (DataTable) | `Arrow Up` / `Arrow Down` |

### Overlay layers

- `Dialog`, `Sheet`, `DropdownMenu`, `Popover`, `ContextMenu`, and
  `Command` palette ship Radix-provided focus traps. Tab cycles within
  the open layer; Escape closes the topmost layer and **returns focus
  to the opener** (the contract enforced by `useRestoreFocus` in
  `lib/utils/a11y.ts`).
- Nested layers stack: a Dialog opened from inside a Sheet returns
  focus to the Sheet, not all the way back to the page.

## ARIA approach

> **Semantic HTML first. ARIA only fills semantic gaps.**

This rule is from the constitution and is non-negotiable. Concretely:

- Native elements (`<button>`, `<a href>`, `<input>`, `<select>`,
  `<dialog>`, `<details>`, `<table>`, etc.) carry their own semantics.
  Do not redundantly add `role="button"` to a `<button>`.
- Use `aria-label` / `aria-labelledby` only when the visible text is
  absent or insufficient (icon-only buttons, decorative wrappers
  around interactive children).
- Use `aria-describedby` for supporting copy (help text, validation
  messages). The `Field` composite wires this automatically.
- Use `role="status"` (polite) for non-blocking updates (toast
  surfaces); use `role="alert"` (assertive) only for genuine error
  conditions that interrupt operator flow.
- Forms use `aria-required="true"` paired with a visible required
  indicator (the `Field` composite handles both — never rely on color
  alone).
- Live regions on the data table (loading / error states) are
  announced via the loading state's `role="status"` (polite by
  default).

## Color-vs-icon affordances

> **Color is never the only signal.** (FR-026)

Every semantic state — success, warning, error, info — ships with:

1. A semantic color from `--color-{success,warning,error,info}-{surface,foreground}`.
2. A canonical icon (CheckCircle2, AlertTriangle, XCircle, Info).
3. Text that names the state.

This is enforced by:

- `<Alert>`, `<Badge>`, `<InlineValidation>`, and the Sonner toast
  surface — each renders the icon automatically when an intent is set.
- The `Badge` primitive's `icon` prop accepts a custom override and
  accepts `false` only when the consumer can prove a non-color affordance is
  already present in the surrounding context.

The visual contract is documented in
[03 — Typography](?path=/docs/foundation-03-typography--docs) and
demonstrated in every primitive's "all states" story.

## Reduced-motion strategy

> **Motion clarifies state; it does not entertain.** (FR-025 / SC-008)

The reduced-motion strategy is layered:

1. **Global CSS rule.** `app/globals.css` has a `@media (prefers-reduced-motion: reduce)`
   block that collapses every animation and transition to ~0.01ms. This
   handles CSS-driven motion across every primitive (Dialog/Sheet open
   transitions, Tabs underline slide, hover transitions, Toast slide-in).
2. **JavaScript-scheduled motion** is gated by the `useReducedMotion()`
   hook in `web/hooks/use-reduced-motion.ts` (T108). The hook subscribes
   to the media query via `useSyncExternalStore` so consumers re-render
   when the OS preference flips.
3. **Recharts series enter/update tweens** are JS-scheduled. The chart
   wrappers (`<ChartLine>`, `<ChartBar>`, `<ChartArea>`) pass
   `isAnimationActive={false}` to each series when `useReducedMotion()`
   returns true. The Playwright spec `tests/e2e/reduced-motion.spec.ts`
   (T110) verifies this end-to-end.

### Authoring guidance

- Default to no motion. Add motion when it expresses **state** (an
  overlay opening, a row revealing). Never add motion as decoration.
- If motion is non-essential, gate it with `useReducedMotion()` or
  with a CSS `transition-*` utility (the global rule covers the latter
  automatically).
- Framer Motion is sanctioned by the constitution but should be used
  sparingly. Most foundation motion is CSS — keep it that way.

## RTL strategy

> **RTL-safe by construction**, even though v1 content is English-only.
> (FR-022b / FR-022d / SC-011)

The RTL surface is enforced statically AND at runtime:

- **Static**: the ESLint rule `busterminal/no-physical-direction-utilities`
  bans `ml-*`, `mr-*`, `pl-*`, `pr-*`, `left-*`, `right-*`, `text-left`,
  `text-right` outside `web/styles/tokens.css`. The audit script
  `pnpm audit:directions` is the CI-side guard.
- **Runtime**: every primitive uses CSS logical properties
  (`margin-inline-start`, `padding-inline-end`, `inset-inline-start`,
  `text-start`, etc.). When `dir="rtl"` is set on the document root,
  layout flips at the engine level — no manual mirror code is needed.
- **End-to-end verification**: the Playwright spec
  `tests/e2e/rtl-smoke.spec.ts` (T111) walks the demo screen with
  `dir="rtl"` in both dark and light themes and asserts no clipping,
  no viewport overflow, and correct anchor positioning for menus,
  popovers, and the Sheet.

### Locale formatters

`web/lib/i18n/format.ts` (T019) provides:

- `formatDate`, `formatTime`, `formatRelativeTime` — wrap
  `Intl.DateTimeFormat` / `Intl.RelativeTimeFormat`.
- `formatDuration`, `formatBytes` — wrap `Intl.NumberFormat`.
- `formatNumber` — direct `Intl.NumberFormat` pass-through.

All formatters resolve the active locale at call time
(`navigator.language` by default; overridable via the `locale` option).
Locale-correct output is verified in `tests/unit/format.test.ts` (T154).

## Failure modes (what to escalate)

Treat each of the following as a defect, not a polish task:

- Any `vitest-axe` violation in a primitive's unit test.
- Any `@storybook/test-runner` violation under `pnpm test:storybook`.
- A primitive that requires a pointing device for any interaction.
- A primitive that signals state with color alone.
- Any hardcoded user-facing string inside a primitive or composite.
- Any physical-direction utility outside `styles/tokens.css`.
- A focus ring removed without a documented, equally-visible replacement.
- Motion that does not respect `prefers-reduced-motion: reduce`.
